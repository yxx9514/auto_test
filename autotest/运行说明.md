## 自动化测试项目 - 运行说明（pytest + allure）

### PyCharm：`from framework` 红线（未解析引用）

本项目顶层包名是 **`framework`** / **`pages`**，目录在 **`autotest/`** 下，不是仓库根目录。

任选其一即可：

1. **已配置**：仓库内 `.idea/transaction_review.iml` 已将 **`autotest`** 标为源码根；用 **File → Invalidate Caches / Restart** 刷新后，红线应消失。
2. **手动**：在 Project 视图里右键 **`autotest` 文件夹** → **Mark Directory as** → **Sources Root**（若同时写 `test_platform`，可把 **`test_platform`** 也标为源码根，便于 `from app ...`）。

仍需在项目里执行 **`pip install -e .`**（在 `autotest` 目录），运行与类型推断才一致。

### 1) 安装（建议独立虚拟环境）

在 `autotest/` 目录执行：

```powershell
py -m venv .venv
.\.venv\Scripts\activate
py -m pip install -U pip
pip install -e .
```

### 2) 运行测试

```powershell
pytest
```

仅跑某一类：

- API：`pytest -m api`
- 关键字：`pytest -m keyword`
- UI：`pytest -m ui`

### 3) 生成 Allure 报告

前提：本机已安装 Allure CLI（`allure` 命令可用）

```powershell
allure serve allure-results
```

### 4) UI 自动化（Playwright，可选）

如果要跑 UI 用例：

```powershell
pip install -e ".[ui]"
playwright install
pytest -m ui
```
